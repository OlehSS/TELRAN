1. git checkout (восстановить файлы из последнего комита)
2. git restore --staged (git reset)
3. git reset HEAD~1
4. git revert
5. git branch - создать ветку, git checkout - для переключения на другую ветку. git checkout -b для создания и одновременного переключения.
6. Отличие в fast-forward merge and non-forward merge в том что в первом случае происходит слияние веток при котором одна не менялась, а во втором случае обе изменились и был создан новый коммит.
7. Конфликт слияния веток возникает когда в разных ветках был изменен один и тот же файл.
Для решения конфликта можно либо отменить слияние командой git merge --abort, либо внести общие изменения в файл, сохранить измененный в обоих ветках и сделать коммит.
8. git branch -d и название ветки.
